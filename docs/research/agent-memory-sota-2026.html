Executive Summary

A systematic review of agent memory systems in April 2026 reveals that memex's hybrid retrieval architecture (z-score fusion of vector + BM25) achieves 90% E2E accuracy on LongMemEval — within 1.4 percentage points of the benchmark leader (Hindsight at 91.4%). The gap is primarily attributable to one missing feature.

Key Finding: Entity extraction at store time + entity overlap boost at retrieval is the single highest-ROI improvement available. It implements ACT-R spreading activation (a 40-year validated cognitive model), closes the gap with Hindsight, enables contradiction detection, and costs 250KB + ~2 hours of implementation.

Second priority: expose memex as an MCP server to transform it from an OpenClaw plugin into a universal memory layer for any agent system.

Research Deep Dives

Iter 1 Landscape Scan + Letta Deep Dive

Letta (formerly MemGPT) separates memory management from conversation into async "sleep-time agents" — validating memex's dreaming concept. Key gap: Letta has self-editing core memory + skill learning; memex has document search + zero infrastructure.

"The gap between 'has memory' and 'does not have memory' is often larger than the gap between different LLM backbones." — March 2026 Agent Memory Survey
Iter 2 Hindsight/TEMPR — How They Hit 91.4%

Hindsight separates memory into 4 networks (World, Bank, Opinion, Observation) and retrieval into 4 signals (vector, BM25, graph traversal, temporal filtering). The 1.4% gap with memex is primarily from the entity graph traversal signal.

Iter 4 A-Mem — Zettelkasten-Inspired Memory (NeurIPS 2025)

A-Mem creates evolving knowledge networks where new memories trigger updates to existing ones. Key idea for memex: memory evolution at store-time (or via dreaming batch).

Iter 5 Forgetting — When Should Memories Die?

Three strategies: temporal decay (have it), usage-based eviction (have it), staleness detection (don't have it). The hard problem: detecting when a high-relevance memory becomes confidently wrong. Solution: contradiction detection at store time using entity overlap.

Iter 6 Procedural Memory / Skill Learning

MACLA (AAMAS 2026) compresses 2,851 trajectories into 187 reusable procedures (15:1). Key insight: separate learning (in memory) from reasoning (in frozen LLM). This is exactly memex's architecture.

New category: category: "procedure" — "how to do X" alongside "what is X".

Iter 7 Memory Compression at Scale

At 10K+ entries, vector search slows. Hierarchical summarization (15:1 compression) is what dreaming reflection produces. MemoryArena finding: systems scoring perfectly on recall "plummet to 40-60%" on agentic tasks.

Iter 8 Cross-Platform Memory via MCP

MCP is the de facto standard for connecting LLMs to tools. Memex as an MCP server = universal memory layer for any agent. Every tool (Claude Code, Cursor, Zed, OpenAI) gets persistent memory by adding one line to .mcp.json.

"MCP is doing for AI integration what REST did for web services."
Iter 9 Temporal Reasoning in Queries

"What happened last week with X?" — memex ignores the temporal constraint. Fix: regex-based detection of "yesterday", "last week", "in March" → convert to date range → WHERE timestamp BETWEEN. Zero LLM cost, handles 80% of temporal queries.

Iter 10 Memory Evaluation Benchmarks

LongMemEval measures recall. MemoryArena (2026) measures whether memory improves task outcomes — and reveals that good recall ≠ good task performance. Memex should add multi-hop + temporal eval questions.

Iter 11 Graph Memory in SQLite

At memex's scale (~2K entries), entity tags outperform graph traversal. SQLite handles graph queries via recursive CTEs up to ~100K nodes. A simple memory_links adjacency table is sufficient if entity tags aren't enough.

Iter 12 xMemory — Semantic Hierarchy + Uncertainty Gating
"Semantic similarity is a candidate-generation signal; uncertainty is a decision signal."

xMemory builds a 4-level hierarchy (messages → episodes → semantics → themes). Retrieval starts at themes, drills down only if uncertainty is high. Result: 29% token reduction + better accuracy. This is what dreaming reflection naturally builds toward.

Iter 13 Multi-Agent Memory

Research frames multi-agent memory as a computer architecture problem (cache coherence, access control). Key insight for memex: store agentId on every memory for provenance. Enable agent-affinity boosting at retrieval.

Iter 14 MemOS — Memory as Operating System

MemOS has an OpenClaw plugin (direct competitor). Novel idea: next-scene prediction — proactively preload memories predicted for the next turn. Like CPU prefetching for agent memory. 60.95% token reduction on LoCoMo.

Iter 15 ACT-R Cognitive Architecture

Memex's scoring pipeline already implements ~70% of ACT-R's activation model. The missing piece: spreading activation (boosting memories sharing context with the query). Entity extraction IS spreading activation. Adding it completes the cognitive model.

activation = vector_score
  * (0.7 + 0.3 * importance)      // base-level
  * recency_factor                  // temporal decay
  * (1 + entity_overlap * 0.15)    // spreading activation (NEW)
  * (1 + recall_frequency * 0.05)  // frequency boost

Dreaming Architecture

Memory consolidation system inspired by human sleep cycles. Three phases, each independently shippable.

Light Sweep
dedup, noise, fragments
→
Deep Sweep
re-score, decay
→
Reflection
learnings (future)

Light Sweep Shipped

Text hash dedup, conversation fragment purge, noise scan via isNoise(). All idempotent, no LLM needed. Runs in ~30ms on 2K entries.

Deep Sweep Shipped

Re-score importance from recall frequency (recalled 5+ times → boost). Decay old never-recalled entries. Ephemeral action log decay. Idempotent.

Reflection Planned

Synthesize learnings from scattered facts. Produce new category: "learning" entries with correction chains. Uses dedicated LLM or client tool.
